New Contracts!
We have added support for several new Flow Contracts! 🎉
  • One of those new contracts is Racing Time — check out their NFT-based next gen racing game built on Flow! (Address: 0×8d4fa88ffa2d9117)
 
{String: AnyStruct} to {String: String? }
We migrated our metadata type in the public repo from a {String: AnyStruct} dictionary to a {String: String? } dictionary for all contracts. This should ensure that all of your token metadata is properly deserialized in our NFT API. We’re still working with the Flow team to build a new metadata standard. Please send us a PR if you find any missing metadata in your contract.

📋  Verification of NFT contracts / collections
Problem: Contract creators can airdrop NFTs into anyone’s wallet. Then when a friend or crypto celeb has hundreds of unverified or spam NFTs, it’s hard to know which NFTs they’ve bought or care about. This is especially painful if you are building an NFT viewer and users see a bunch of unexpected NFTs.

Solution: Still brainstorming, but considering:
(1) Using existing "verification" statuses on OpenSea and Etherscan
(2) Develop our own on-chain verification or spam indicator.

If this has been an issue for you too, don’t forget to leave feedback here — it will help us prioritize!

🍬  NFT Minting API
We’ve gotten requests to create a minting service or API. We are in the earliest phases of exploring this idea, and would love your feedback.

Are you looking for a minting API? What would need from a minting API? Are you using any currently? Please reach out through this form if this is something you are interested in.

⌛ Addressing IPFS timeouts and rate limits
Since we make a whole lot of requests on the IPFS gateway, we run into problems with about 1% of requests. This is fixable with retries, but we’re brainstorming other solutions.

In the meantime if IPFS is a problem for you please reach out via Discord! We may have short term solutions to help.

👀 Monitoring development of new NFT standards
We are keeping on eye on several new EIPs that propose improvements to existing NFT standards. We will support any that gain significant traction. We’re currently monitoring:This is fixable with retries, but we’re brainstorming other solutions.
  • ERC721A an extension of ERC721, intended to lower gas costs by allowing minting of multiple NFTs in a single transaction. Read more here.
  • EIP 1496 which proposes that NFTs emit a special ‘MetadataUpdate’ event when the token metadata is updated. Particularly useful for gaming NFTs where player stats might update fairly often.
  • EIP 4910 which proposes an on-chain NFT royalty standard to avoid too much centralization of royalty structures.
